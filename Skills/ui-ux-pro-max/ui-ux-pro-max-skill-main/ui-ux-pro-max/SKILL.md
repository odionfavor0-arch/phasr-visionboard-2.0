# UI/UX Pro Max

## Overview

Use this skill to make user-facing interfaces feel intentional, polished, accessible, responsive, and emotionally clear. Preserve the existing product structure and logic unless the user explicitly asks for a larger redesign.

## Core Workflow

1. Identify the screen, flow, component, or state the user wants improved.
2. Read the existing UI patterns before making changes: layout primitives, spacing scale, component conventions, colors, typography, icons, responsive breakpoints, and interaction states.
3. Improve mobile behavior first, then desktop and wide-screen behavior.
4. Keep edits scoped to interface quality unless the user asks for product or architecture changes.
5. Verify visual behavior after implementation when a runnable app or page is available.

## Design Priorities

Prioritize in this order:

1. Visual hierarchy
2. Spacing consistency
3. Responsive behavior
4. Interaction states
5. Accessibility
6. Emotional UX
7. Clean component balance
8. Premium but simple presentation

## Core Rules

- Do not redesign the whole app structure unless requested.
- Preserve existing product logic, routing, data flow, and core user flows.
- Prioritize clarity, spacing, hierarchy, consistency, responsiveness, and interaction feedback.
- Make the interface feel intentional, calm, premium, and emotionally supportive.
- Keep tap targets comfortable and accessible.
- Maintain strong text contrast and visible focus states.
- Use a consistent spacing rhythm across sections and components.
- Prefer familiar icon components for controls when the project already has an icon library.
- Avoid clutter, visual noise, unnecessary containers, and generic AI-looking layouts.
- Respect the existing product language unless the user asks for a stronger visual shift.

## Interaction Rules

- Make buttons and controls clearly tappable or clickable.
- Make active, selected, disabled, loading, empty, success, error, and done states visually intentional.
- Keep hover states subtle and smooth; do not rely on hover for required actions.
- Keep important actions visible without overwhelming the screen.
- Avoid awkward empty space, especially on mobile.
- Prevent text, controls, fixed bars, and cards from overlapping at small widths.
- Use stable dimensions for controls, counters, grids, toolbars, and fixed-format UI so state changes do not shift layout.

## Accessibility Checks

- Ensure touch targets are at least 44px by 44px where practical.
- Keep body text readable and avoid tiny labels that become illegible on mobile.
- Preserve visible keyboard focus states.
- Provide accessible labels for icon-only controls.
- Do not communicate meaning by color alone.
- Keep contrast strong for text, icons, borders, disabled states, and feedback states.
- Respect reduced-motion preferences for decorative or nonessential animation.

## Project Focus

For the Visionboard project, prioritize refinement of:

- showroom experiences
- feed systems
- ranking layouts
- check-in interactions
- dashboards and room surfaces
- mobile polish
- desktop responsiveness
- spacing consistency
- emotional UX

## Delivery Checklist

Before finishing UI/UX work, confirm the interface has no obvious mobile overflow, no clipped text, no incoherent overlap, no low-contrast essential text, and no controls that look interactive but do not respond.
